Tech Support Guides


Obtaining System Information

  1. From the Start Menu, type "run" and hit Enter.
  2. In the prompt, type "msinfo32" and click OK

    This will run the System Information tool, which provides an extremely in-depth overview of the system.

  3. From the File menu, Save the file. Use your name as the filename. Using your name isn't mandatory, just makes things a bit easier.
  4. Make sure the file extension (Save As Type) is set for "System Information File .NFO".
  5. Save it to your Desktop so that it's easy to find.

BSOD (Blue Screen Of Death)

Although there are countless causes for a BSOD, Windows is usually able to document the reason it crashed. Two ways in which it can do this is:

  • A STOP error code

    This complicated-looking code is shown on the blue screen, about 10 digits in length, starting with "0x". The number of possible STOP codes is extensive to say the least, but a BSOD will only show one, and provides a good starting point as to why the system has crashed. Don't worry about this. I saw it in the screenshot you sent me, so I have this info already.

  • Obtaining and reading a minidump(.dmp) file

    Whenever Windows crashes and displays a BSOD, it shuts itself down. However, the blue screen itself can vary on how long it is displayed. Sometimes it's visible for a split second, other times it may never actually disappear. However, Windows will always generate and store a "minidump" file in the C:\Windows folder. This file/folder may be hidden, so it needs to be located manually (usually a search will find it pretty fast). This file is much more detailed about what caused the crash, but still very hard to interpret depending on the cause. It may show everything, it may show nothing. This is the second file I want to look at.

    1. From the Start Menu, click on "Computer".
    2. Click on Local Disk (C:)
    3. Type "minidump" in the Search bar (upper right).
    4. The results should show a folder called Minidump. Open this folder. These are the dump files. How many depends on how many times Windows has blue-screened, but they all should end in .dmp
    5. Find the one with the latest date. Copy and paste it to the Desktop.
